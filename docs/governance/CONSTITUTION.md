# The Renegade Constitution

> **Document role: specification.** This document defines enduring design
> principles and intended direction. It does not describe the full current
> implementation; begin with `README.md` for current status.

> "Build the spine before teaching the body to move."

---

# Purpose

Renegade exists to explore whether a deterministic symbolic cognitive architecture can grow into a genuinely general reasoning system.

The objective is not to maximize an ARC benchmark.

The objective is to build an architecture capable of developing reusable reasoning.

ARC is a proving ground—not the destination.

---

# Prime Directive

Never sacrifice architectural integrity for benchmark performance.

A temporary score increase is meaningless if it damages generality.

Every decision should make the architecture more reusable, more understandable, and more capable of transfer.

---

# First Principles

The following principles are non-negotiable.

## 1. Determinism

Identical inputs produce identical outputs.

There is no hidden randomness.

There is no stochastic reasoning.

Every decision must be reproducible.

---

## 2. Structure Before Intelligence

Every capability must have a place before it has an implementation.

Nothing exists in isolation.

Everything belongs somewhere.

---

## 3. Everything Has Relationships

Knowledge is not a list.

Knowledge is a hierarchy.

Every concept knows:

- what it depends on
- what depends on it
- where it belongs
- why it exists

---

## 4. Learning Produces Reusable Behavior

Learning is not:

- metadata
- counters
- rewards
- cached outputs
- statistics

Learning creates reusable capability.

If nothing reusable exists afterwards, nothing was learned.

---

## 5. Memory Advises

Memory never commands.

Past experience may suggest.

Current evidence decides.

---

## 6. Capabilities Stay Dormant

The system should never activate everything.

Capabilities exist like tools in a workshop.

Retrieve what is needed.

Leave everything else asleep.

---

## 7. Simplicity Wins

If two architectures solve the same problem:

prefer

- fewer concepts
- fewer assumptions
- fewer moving parts
- clearer reasoning

Complexity requires justification.

---

## 8. Abstraction Is Compression

Repeated successful reasoning should become simpler.

The architecture should compress itself over time.

Not expand forever.

---

## 9. Pruning Is Intelligence

Keeping everything active is not intelligence.

Unused branches become archived.

Repeated concepts become unified.

Organization is part of reasoning.

---

## 10. Explainability Is Mandatory

Every decision should have a reason.

Every hypothesis should have evidence.

Every learned capability should have provenance.

If the architecture cannot explain itself, it should not trust itself.

# The Executive

The executive is not the intelligence.

The executive is the organizer.

Its responsibilities are:

- observe
- retrieve
- decide
- execute
- criticize
- repair
- consolidate

Nothing more.

---

# Memory

Memory exists to reduce future work.

Not to replace reasoning.

Remember structure.

Forget clutter.

Archive history.

Promote concepts.

# What Renegade Is Not

Renegade is not:

- an LLM
- a neural network
- a benchmark optimizer
- a random search engine
- a giant rule database
- an ARC-specific solver

---

# What Success Looks Like

Success is not measured by score alone.

Success is measured by increasing ability to:

- understand unfamiliar problems
- organize knowledge
- create reusable abstractions
- transfer previous reasoning
- simplify itself
- explain itself

---

# Engineering Principles

Every module should answer:

Why do I exist?

What depends on me?

What do I depend on?

Could I be simpler?

Can I be composed?

Can I transfer?

---

# The Final Rule

Never build for today's benchmark.

Always build for tomorrow's unknown problem.

If the architecture becomes capable enough, benchmarks become consequences rather than goals.
